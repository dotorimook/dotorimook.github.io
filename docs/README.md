# 작업 기록 문서 (Work Records)

이 폴터는 프로젝트의 주요 변경사항과 작업 이력을 기록하는 공간입니다.

## 폴터 구조

```
docs/
├── README.md                 # 이 파일
├── records/                  # 작업 로그 저장소
│   └── YYYY-MM-DD-작업명.md  # 날짜별 작업 기록
└── guides/                   # (예정) 가이드 문서
    └── astro-seo-guide.md
```

## 문서 네이밍 규칙

작업 기록 파일은 다음 형식을 따릅니다:

```
YYYY-MM-DD-작업명.md
```

**예시**:
- `2025-02-18-seo-improvements.md`
- `2025-03-01-design-system-update.md`
- `2025-03-15-performance-optimization.md`

## 기록 항목

각 작업 기록 문서에는 다음 내용이 포함됩니다:

1. **작업 개요**: 작업일, 작업자, 목적
2. **작업 항목**: 세부 작업 내용 및 변경사항
3. **변경 파일**: 수정된 파일 목록
4. **검증 사항**: 테스트 및 확인 내용
5. **다음 단계**: 후속 작업 또는 권장사항
6. **참고 자료**: 관련 링크 및 문서

## 작성 가이드

### 새 작업 기록 작성 시

1. `docs/records/` 폴터에 새 파일 생성
2. 파일명: `YYYY-MM-DD-작업명.md`
3. 템플릿 활용 (아래 참고)

### 템플릿

```markdown
# [작업명]

## 작업 개요
- **작업일**: YYYY-MM-DD
- **작업자**: [이름]
- **목적**: [작업 목적 설명]

## 작업 항목

### 1. [작업 제목] ✅

**파일**: `[파일 경로]`

**변경 내용**:
```[언어]
[코드 또는 변경사항]
```

**목적**: [왜 이 변경이 필요한지]

## 검증 완료 사항
- [ ] [체크리스트 항목]

## 다음 단계
- [ ] [후속 작업]

## 참고 자료
- [링크 설명](URL)
```
